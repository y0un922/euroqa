## 1. 需求

### 背景
当前前端会把 LLM 正文中的规范引用直接渲染成较长的内联芯片，命中不到现有 `sources` 时视觉上又会退化成弱提示文本，导致引用和正文混在一起，阅读节奏被打断，也不够像常见 LLM 产品中的“轻量可点来源锚点”。

### 目标
- 保留现有 `linkifyReferenceCitations` 引用识别链路，不重做 Markdown 解析。
- 将正文中的长文本引用改为更轻量的编号锚点，点击后继续联动右侧证据与溯源面板。
- 在来源列表中显示与正文一致的编号，降低用户在正文与来源区之间来回比对的成本。
- 为 `EN 1990:2002 · A1.2.1(4)` 这一类附件条款引用补最小回归测试，避免再次退化。

### 约束条件
```yaml
时间约束: 本次只处理当前前端引用渲染链路，不扩展后端协议。
性能约束: 保持现有 ReactMarkdown + linkify 链路，不引入重型解析器。
兼容性约束: 继续兼容已存在的 reference:// 与 citation:// 内部链接协议。
业务约束: 右侧证据与溯源面板的数据结构和交互入口不变。
```

### 验收标准
- [ ] `linkifyReferenceCitations` 对 `EN 1990:2002 · A1.2.1(4)` 这类 plain citation 有回归测试覆盖。
- [ ] 正文内引用改为明显区别于正文的紧凑编号锚点，命中来源时可点击联动证据面板。
- [ ] “引用来源”列表显示与正文一致的编号，便于映射。
- [ ] 前端测试通过，且不影响普通外链 Markdown 渲染。

---

## 2. 方案

### 技术方案
保留 `frontend/src/lib/citations.ts` 负责把规范引用文本改写为内部链接，再新增一个轻量级引用展示辅助模块，根据 `referenceId + references[]` 生成正文内引用锚点所需的编号、标题和状态。`MainWorkspace.tsx` 中的 Markdown `a` 渲染器不再直接显示完整条款文本，而是输出更接近 GPT / Gemini 的紧凑编号按钮；未命中来源时则降级为中性编号提示。底部“引用来源”按钮同步展示编号徽标，以保持正文锚点和来源区一致。

### 影响范围
```yaml
涉及模块:
  - frontend.components.MainWorkspace: 正文引用渲染与来源列表样式调整
  - frontend lib citations: 引用匹配回归测试补强
  - frontend lib inlineReferences: 新增正文引用编号展示逻辑
预计变更文件: 5
```

### 风险评估
| 风险 | 等级 | 应对 |
|------|------|------|
| 编号锚点与来源列表顺序不一致 | 中 | 统一基于 `references` 数组顺序生成编号，并补单测 |
| 未命中引用提示过弱或过强 | 低 | 为未命中引用保留中性样式和完整 tooltip |
| 普通 Markdown 外链被误伤 | 低 | 保持 `href` 前缀判断，仅处理内部引用协议 |

## 4. 核心场景

> 执行完成后同步到对应模块文档

### 场景: 正文内点击引用查看证据
**模块**: frontend.components.MainWorkspace
**条件**: LLM 回答正文中包含已识别的规范引用，且消息附带 `sources`
**行为**: 用户点击正文中的引用编号锚点
**结果**: 右侧证据与溯源面板切换到对应来源，正文与来源列表编号一致

### 场景: 正文中出现未命中引用
**模块**: frontend.components.MainWorkspace
**条件**: LLM 输出了规范引用，但当前返回的 `sources` 中没有对应来源
**行为**: 前端将该引用渲染为中性编号提示，并提供完整 tooltip
**结果**: 用户能区分该文本是引用而非正文，但不会误以为可点击跳转

## 5. 技术决策

> 本方案涉及的技术决策，归档后成为决策的唯一完整记录

### citation-anchor-ui#D001: 正文引用改为轻量编号锚点
**日期**: 2026-03-28
**状态**: ✅采纳
**背景**: 当前正文直接嵌入完整条款标签，视觉过重，且长条款会明显打断阅读节奏。
**选项分析**:
| 选项 | 优点 | 缺点 |
|------|------|------|
| A: 轻量编号锚点 | 阅读负担低，接近 GPT / Gemini，正文和来源区可建立稳定映射 | 需要额外生成编号和 tooltip |
| B: 保留长文本芯片，仅调颜色边框 | 改动最小 | 仍然打断正文阅读，提升有限 |
**决策**: 选择方案 A
**理由**: 这次问题的核心不是“看不见”，而是“正文内引用过重且不顺手”。编号锚点能把视觉重点还给正文，同时继续保留来源可追溯性。
**影响**: 影响 `MainWorkspace` 的 Markdown 链接渲染方式，以及底部“引用来源”列表的呈现形式
